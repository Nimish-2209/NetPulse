# Git Workflow

Use small, readable commits that match project milestones.

## Suggested Branches

- `main`: stable application code.
- `feature/auth`: authentication and team membership.
- `feature/services`: monitored service CRUD.
- `feature/worker`: uptime check worker.
- `feature/realtime`: Socket.IO updates.
- `feature/incidents`: incident lifecycle.

## Commit Style

Examples:

- `chore: scaffold monorepo structure`
- `feat(auth): add JWT login flow`
- `feat(worker): record uptime check results`
- `docs: add API route plan`
